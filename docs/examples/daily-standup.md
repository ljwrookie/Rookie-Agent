# Example: Daily Standup

This example demonstrates how Rookie Agent can help generate daily standup reports from your git activity.

## Scenario

At the end of the day, you need to provide a standup update. Instead of manually reviewing your commits, Rookie Agent analyzes your git history and generates a summary.

## Prerequisites

- Rookie Agent CLI installed
- A git repository with commits

## Setup

Let's create a sample project with commits:

```bash
mkdir -p /tmp/standup-example
cd /tmp/standup-example
git init

# Configure git for the example
git config user.email "dev@example.com"
git config user.name "Developer"

# Create initial structure
mkdir -p src/components src/utils tests

# Commit 1: Morning work
cat > src/utils/api.js << 'EOF'
export async function fetchUser(id) {
  const response = await fetch(`/api/users/${id}`);
  return response.json();
}
EOF
git add .
git commit -m "feat: add user API client

- Implement fetchUser function
- Add error handling for 404 responses"

# Commit 2: Mid-day work
cat > src/components/UserProfile.jsx << 'EOF'
import { useEffect, useState } from 'react';
import { fetchUser } from '../utils/api';

export function UserProfile({ userId }) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    fetchUser(userId).then(user => {
      setUser(user);
      setLoading(false);
    });
  }, [userId]);

  if (loading) return <div>Loading...</div>;
  return <div>{user.name}</div>;
}
EOF
git add .
git commit -m "feat: add UserProfile component

- Display user information
- Add loading state
- Fetch data on mount"

# Commit 3: Afternoon work
cat > tests/UserProfile.test.jsx << 'EOF'
import { render, screen } from '@testing-library/react';
import { UserProfile } from '../src/components/UserProfile';

test('renders user name', async () => {
  render(<UserProfile userId="123" />);
  expect(await screen.findByText('John Doe')).toBeInTheDocument();
});
EOF
git add .
git commit -m "test: add UserProfile tests

- Test rendering with user data
- Mock API calls"

# Commit 4: Bug fix
cat > src/utils/api.js << 'EOF'
export async function fetchUser(id) {
  const response = await fetch(`/api/users/${id}`);
  if (!response.ok) {
    throw new Error(`Failed to fetch user: ${response.status}`);
  }
  return response.json();
}
EOF
git add .
git commit -m "fix: handle API errors properly

- Throw error on non-2xx responses
- Fixes issue where 500 errors were silently ignored"
```

## Using Rookie Agent

### 1. Generate Standup Report

```bash
cd /tmp/standup-example
rookie standup
```

Or in an interactive session:

```
> Generate my daily standup report for today
```

### 2. Rookie Agent's Analysis

Rookie Agent will:

1. **Query** git log for today's commits
2. **Analyze** commit messages and changed files
3. **Categorize** work by type (features, fixes, tests)
4. **Generate** a formatted standup report

### 3. Expected Output

```markdown
## Daily Standup Report - 2026-04-23

### Summary
- 4 commits
- 3 files created, 1 file modified
- Focus: User profile feature

### Completed Work

**Features**
- ✅ Implemented user API client (`src/utils/api.js`)
- ✅ Created UserProfile React component with loading states

**Testing**
- ✅ Added unit tests for UserProfile component

**Bug Fixes**
- ✅ Fixed API error handling (500 errors now properly thrown)

### In Progress
- User profile feature (80% complete)

### Blockers
- None

### Next Steps
- Add user profile editing functionality
- Implement avatar upload
- Review with team tomorrow

### Code Stats
```
Files changed: 4
Insertions: 85
Deletions: 12
Tests added: 1
```

---
*Generated by Rookie Agent*
```

### 4. Customization

You can customize the report:

```
> Generate standup for the last 3 days
> Show only commits related to authentication
> Include code statistics
```

## CLI Options

```bash
# Today's standup
rookie standup

# Specific date range
rookie standup --since "2026-04-20" --until "2026-04-23"

# Specific author
rookie standup --author "Developer"

# Output formats
rookie standup --format markdown  # Default
rookie standup --format json      # Machine readable
rookie standup --format slack     # Slack-formatted

# Save to file
rookie standup --output standup.md
```

## Integration with Calendar

Rookie Agent can also include calendar events:

```
> Include my calendar events in the standup
```

Output:
```markdown
### Calendar
- 10:00 AM - Sprint planning (completed)
- 2:00 PM - Code review with team (completed)
- 4:00 PM - 1:1 with manager (upcoming)
```

## Weekly Summary

For weekly reports:

```bash
rookie standup --weekly
```

Generates:
```markdown
## Weekly Summary

### Accomplishments
- Completed user profile feature
- Fixed 3 critical bugs
- Added 15 new tests

### Challenges
- API latency issues on Tuesday
- Resolved with caching implementation

### Goals for Next Week
- Implement user settings page
- Optimize database queries
```

## Configuration

Create `.rookie/standup-config.json`:

```json
{
  "format": "markdown",
  "sections": [
    "summary",
    "completed",
    "in-progress",
    "blockers",
    "next-steps"
  ],
  "includeStats": true,
  "includeCalendar": false,
  "timezone": "Asia/Shanghai"
}
```

## Automation

Add to your shell profile for automatic standup at end of day:

```bash
# .zshrc or .bashrc
alias standup='rookie standup --output ~/standups/$(date +%Y-%m-%d).md'
```
